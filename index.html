User Locations ¶

Create a new location  ¶

Create a new location
POST/api/user/location

Allows an authenticated user to create a new location and save it in her personal list.

Locations are expected to be created as copies of other locations, with additional details such as special instructions for the driver.

Example URI

POST /api/user/location

Here Service ¶

What is here?  ¶

What is here?
GET/api/here{?loc}

Returns the geolocalized information for a given point in a single request.

Information provided by Here might be available from other endopoints using several requests. The purpose of here is to reduce network latency.

An optional date and time can be indicated. This is presently designed to be used with reservation requests in order to check availability of vehicle types on the given date and time.

Example URI

GET /api/here?loc=40.345041, -1.1185173
URI Parameters
HideShow
loc
string (required) Example: 40.345041, -1.1185173

A pair of latitude, longitude coordinates

start_at
string (optional) Example: 2020-12-31 23:59

A datetime value in the local timezone. Information returned will be valid for the indicated time, or now if not provided.

Places Service ¶

Get suggestions  ¶

Get suggestions
GET/api/places/autocomplete{?q}

Retrieves a list of suggestions based on partial text input given on the q parameter

The response will be an array of Suggestions

A Suggestion object is a presentation-oriented object with pre-formatted values. It is tailored for a two line display, with a highlighted title and a dimmed subtitle below.

An id is provided so it can be inflated to a full location object.

Example URI

GET /api/places/autocomplete?q=Baraj

Place retrieval ¶

Place retrieval endpoint return a single Location object.

A Location object contains all the information for displaying about a place in an structured format, allowing for custom presentations.

It also includes aditional information useful for requesting journeys from it (currently, the actual coordinates and the saved instructions to the driver)

Locations can have different sources, both shared (from Google Places or Cabify) and personal (saved by the User or Client).
